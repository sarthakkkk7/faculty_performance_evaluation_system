## 🚀 STEP-BY-STEP SETUP (Do these in ORDER)

### STEP 1 — Install Python (if not installed)
Download from: https://www.python.org/downloads/
- Choose Python 3.10 or 3.11
- ✅ Check "Add Python to PATH" during install

### STEP 2 — Create Supabase Account (Free)
1. Go to: https://supabase.com → Sign up (free)
2. Click "New Project" → Give it a name → Set a database password
3. Wait 2 minutes for project to start
4. Go to **Settings → API** and copy:
   - **Project URL** (looks like: `https://abcdefgh.supabase.co`)
   - **Anon/Public Key** (long string starting with `eyJ...`)

### STEP 3 — Setup the Database
1. In Supabase, go to **SQL Editor** (left sidebar)
2. Click **"New Query"**
3. Open the file `SCHEMA_V3.sql` from this project
4. Paste ALL the contents into Supabase SQL Editor
5. Click **"Run"** — you should see "Setup complete!"

### STEP 4 — Configure Environment Variables
1. Copy `.env.example` to `.env`:
   ```
   cp .env.example .env
   ```
2. Open `.env` and fill in:
   ```
   SUPABASE_URL=https://your-project.supabase.co
   SUPABASE_KEY=your-anon-key-here
   SECRET_KEY=any-random-string-like-abc123xyz789
   ```

#### Optional: Email Setup (for real OTP emails)
If you want real OTP emails via Gmail:
1. Enable 2-Factor Authentication on your Gmail
2. Go to: https://myaccount.google.com/apppasswords
3. Create an "App Password" for "Mail"
4. Add to `.env`:
   ```
   SMTP_USER=your-gmail@gmail.com
   SMTP_PASS=xxxx-xxxx-xxxx-xxxx   ← the 16-char app password
   SMTP_FROM=your-gmail@gmail.com
   ```
> ⚠️ **Without email config, OTP will print to the console** — this is fine for development!

### STEP 5 — Install Python Dependencies
Open a terminal/command prompt in the project folder and run:
```bash
pip install -r requirements.txt
```
(This may take 2-3 minutes)

### STEP 6 — Train the ML Model (One-time)
```bash
python train_ml_model.py
```
This creates: `ml_model.pkl`, `scaler.pkl`, `label_encoder.pkl`, `model_metadata.json`

### STEP 7 — Import Faculty Data (One-time)
```bash
python import_data.py
```
This imports all faculty and performance data from the CSV file into Supabase.

### STEP 8 — Start the Application
```bash
python app.py
```
You should see:
```
✓ Supabase connected
✓ ML Model loaded
🎓 FACULTY PERFORMANCE SYSTEM v3.0
🌐 Server: http://localhost:5000
```

### STEP 9 — Open in Browser
Go to: **http://localhost:5000**

## 📝 How Registration Works

1. Click **"Register here"** on the login page
2. Fill in your details (name, username, email, password, role)
3. Click **"Send Verification Code"**
4. **If email is configured:** Check your email for the 6-digit OTP
5. **If email is NOT configured (dev mode):** The OTP prints in your terminal console
6. Enter the 6-digit OTP on the verification screen
7. Your account is created — you can now login!

## ❓ Troubleshooting

### "SUPABASE_URL and SUPABASE_KEY must be set"
→ You haven't created the `.env` file. Copy `.env.example` to `.env` and fill in your credentials.

### "ModuleNotFoundError: No module named 'supabase'"
→ Run: `pip install -r requirements.txt`

### "ML model not loaded"
→ Run: `python train_ml_model.py` first

### "No performance data"
→ Run: `python import_data.py` to load the dataset

### Login says "Please verify your email first"
→ If you created users directly in Supabase, run this SQL:
`UPDATE users SET email_verified = TRUE;`

### OTP not arriving by email
→ In dev mode (no SMTP), check your **terminal console** — the OTP is printed there!
→ For Gmail: make sure you used an **App Password**, not your real Gmail password.

### Port 5000 already in use
→ Change the port: `app.run(port=5001)` in `app.py` last line
